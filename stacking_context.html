Stacking context is a concept in CSS that determines the order in which elements are stacked and displayed on a web page. or we can say that which container is display top on the another conatiner.. When an element is positioned on a web page, it forms a stacking context, which is a hierarchical order of its position relative to other elements. This hierarchy determines which elements will be on top or bottom of other elements when they overlap.

Stacking context is determined by the following rules:

  • Each stacking context is self-contained, meaning that the parent of an element in a stacking context does not affect the stacking order of its child elements.
  • The z-index property can only be applied to elements in the same stacking context.
  • The order of the elements in the HTML document affects their stacking order, with later elements stacking on top of earlier elements.

Elements that create a new stacking context include those with a position value of absolute, relative, fixed, or sticky, as well as elements with a transform, opacity, or filter property. These elements will be positioned in front of elements that do not create a new stacking context, even if they have a lower z-index value.
